1.8 The answers for 1.5-1.7 are the same, after my research, I found that in JavaScript, functions are "hoisted" to the top of their scope. This means that even if you call a function before it is defined, the JavaScript engine knows that it will be defined later in the code, so it allows the call to go through. This is why it is possible to call a function before it is defined in JavaScript. The function definition is processed first by the JavaScript engine, and then the rest of the code is executed. This allows you to use a function before it is defined, even if it is called before its declaration.
